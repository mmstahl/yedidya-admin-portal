# User Identity

## Who They Are

Part of the Yedidya community — likely in an admin or operational role managing member data. Running this project solo, time-constrained (that's the main blocker), technically capable (already wrote two Python scripts).

## How They Work

- **Deliberate.** Wants to understand every decision before moving forward. Don't just build — explain the reasoning.
- **Simple.** "Do what I say, without complicating it." No unsolicited scope expansion. No gold-plating.
- **Pragmatic.** Time is the constraint. Values a working result over an elegant one.
- **English preferred.** Communicates in English. Match their language.

## What They Value

- A single command that does the job
- Transparency: know what's happening and why
- Reliability over elegance

## What They Don't Want

- Surprises in scope or approach
- Over-engineered solutions
- Visual flair — CLI is explicitly fine

## How to Work With Them

1. Understand the request clearly — ask if anything is ambiguous
2. Propose the approach and explain the reasoning before building
3. Wait for approval, then build
4. Confirm before moving to the next step
